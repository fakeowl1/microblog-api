# Functional вимоги до системи

## 1. Управлінням профілем користувача 

- FR1: Реєстрація нового користувача
- FR2: Автентифікація користувача
- FR3: Перегляд профілю та редагування власного профілю
- FR4: Видалення облікового запису

## 2. Управління контентом (пости) 

- FR5: Створення нового поста
- FR6: Перегляд певного посту
- FR7: Перегляд власної стрічки
- FR8: Редагування власного поста
- FR9: Видалення власного поста

## 3. Управління соціальними зв'язками

- FR10: Надіслати запит на дружбу
- FR11: Прийняття/відхилення запиту на дружбу
- FR12: Видалити користувача зі списку друзів
- FR13: Перегляд списку друзів та їхніх профілів

## 4. Взаємодія з контентом

- FR14: Можливість лайкнути/дизлайкнути пост
- FR15: Можливість залишити комментар під постом
- FR16: Можливість лайкнути/дизлайкнути комментар
- FR17: Можливість відповісти на комментар

## 5. Системні функції

- FR18: Пошук користувача за іменем
- FR19: Сповіщення про нові лайки, дизлайки, комментарі, запити в друзі, тощо

# Non-functional вимоги до системи

## 1. Продуктивність

- NFR1: Час завантаження сторінки менше ніж 2 секунд
- NFR2: Час відгуку сервера менше ніж 500мс
- NFR3: Система повинно оброблювати 10,000 активних користувачів

## 2. Безпека

- NFR4: Всі паролі повинні бути захешовані з використанням алгоритму bcrypt
- NFR5: Авторизований користувач може виправляти лише свій власний пост/профіль
- NFR6: Система повинна мати логування у форматі json всіх критичних помилок для діагностики проблем

## 3. Зручність

- NFR7: Інтерфейс має бути зрозумілим для користувачів з різним рівнем технічної підготовки
- NFR8: Інтерфейс повинен бути адаптивним і корректно відображатися на мобільних пристроях, планшетах та пк

## 4. Надійність 

- NFR9: Частота критичних збоїв системи не повинна перевищувати 1 раз на 12 місяців
- NFR10: Система має бути доступною 99.9% часу

## 5. Розширюваність
- NFR11: Кодова база повинна мати повну документацію
